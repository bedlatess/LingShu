## 任务一 · 修复 Tailwind 完全失效（P0，当前最严重问题）

**现象**：用户端截图里所有卡片变成横贯全屏的扁条、按钮变成整行灰长条、元素挤成一团、无圆角无间距。

**根因已定位**：装的是 Tailwind v4（`package.json`：`tailwindcss@^4.1.18` + `@tailwindcss/postcss`），但 `src/styles.css` 用的是 v3 老语法 `@tailwind base/components/utilities`，且还留着 v3 的 `tailwind.config.ts`。v4 引擎扫不到类，静默产出空 CSS，所以所有 `className` 全部失效。

> ⚠️ 这正是上一轮"`npm run build` 通过"却没发现的原因——**Tailwind 配错不报错，只产出空样式**。必须在浏览器里看渲染结果，不能只看构建退出码。

**二选一，彻底统一版本，不要混用：**

**方案 A（推荐，保持 v4）**
- `src/styles.css` 把三行 `@tailwind` 指令换成：`@import "tailwindcss";`
- 把 `tailwind.config.ts` 里的主题（颜色/圆角/字体/阴影）迁移到 CSS 的 `@theme` 块
- 删除或停用 `tailwind.config.ts`（v4 不读它）
- 确认 `@tailwindcss/postcss` 在 `postcss.config.js` 中正确挂载

**方案 B（降级 v3）**
- 卸载 v4，装 `tailwindcss@3` + `postcss` + `autoprefixer`
- `postcss.config.js` 改回 `tailwindcss: {}` + `autoprefixer: {}`
- 保留现有 `@tailwind` 写法和 `tailwind.config.ts`

**验收**：`npm run dev` 后在浏览器真实截图 dashboard / usage / api-keys 三页，必须看到：
- 卡片有圆角 + 边框 + 留白
- 按钮是正常尺寸按钮（不是整行长条）
- StatCard 是网格卡片排列
- 深色主题 + 主色生效

截图存到 `docs/evidence/` 覆盖旧图。

## 任务二 · 钱相关路径真实跑通（不用 Docker）

本机无 Docker/psql/redis **不是跳过验证的理由**。用进程内集成测试，无需任何外部服务：

- **Postgres**：用 `github.com/fergusstrange/embedded-postgres`（自动下载真实 PG 二进制并在进程内启动）
- **Redis**：用 `github.com/alicebob/miniredis/v2`（内存版 Redis，兼容现有 redis 客户端）
- **上游**：用 `net/http/httptest` 起一个假 OpenAI 服务，返回带 `usage` 的固定响应

写一组集成测试（放 backend，如 `internal/service/billing_integration_test.go`），针对真实迁移后的库，端到端断言以下四条钱路径的数字正确：

1. **token 模型**：调用 `/v1/chat/completions` → 校验 `charge = base_cost × rate_multiplier`，`users.balance` 正确扣减，`balance_ledger` 和 `gateway_requests` 各写一条且金额一致
2. **per_call 模型（文生图）**：走 per_call 分支，`charge = price_per_call × n × 倍率`，不被当 token 计 0
3. **管理员手动充值**：调 `POST /admin/users/{id}/balance` → 余额增加 + 写 `admin_grant` 账本
4. **余额不足**：预占阶段返回 **402**，且不产生扣费

另外补一个**并发测试**：同一用户余额仅够 N 次，并发 50 次请求，断言恰好成功 N 次、余额不为负。

**验收**：`go test ./...` 全绿，把这几个集成测试的运行输出（PASS + 关键断言）贴出来。
